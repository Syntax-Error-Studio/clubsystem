**关联映射核心代码**（SignupMapper.xml）：

```xml
<!-- ★ 关键：包含关联User和Activity对象的ResultMap定义 -->
<resultMap id="signupWithAllResultMap" type="Signup">
    <id column="s_id" property="id"/>
    <result column="s_user_id" property="userId"/>
    <result column="s_activity_id" property="activityId"/>
    <result column="s_signup_time" property="signupTime"/>
    <result column="s_status" property="status"/>
    
    <!-- 一对一关联：Signup -> User -->
    <association property="user" ofType="User">
        <id column="u_id" property="id"/>
        <result column="u_username" property="username"/>
        <result column="u_real_name" property="realName"/>
        <result column="u_student_no" property="studentNo"/>
        <!-- ... 其他字段 ... -->
    </association>
    
    <!-- 一对一关联：Signup -> Activity -->
    <association property="activity" ofType="Activity">
        <id column="a_id" property="id"/>
        <result column="a_title" property="title"/>
        <result column="a_location" property="location"/>
        <result column="a_start_time" property="startTime"/>
        <!-- ... 其他字段 ... -->
    </association>
</resultMap>

<!-- ★ 关键：使用JOIN实现关联查询 -->
<select id="getSignupById" parameterType="int" resultMap="signupWithAllResultMap">
    SELECT
        s.id s_id, s.user_id s_user_id, s.activity_id s_activity_id, ...,
        u.id u_id, u.username u_username, u.real_name u_real_name, ...,
        a.id a_id, a.title a_title, a.location a_location, ...
    FROM signup s
    LEFT JOIN `user` u ON s.user_id = u.id
    LEFT JOIN activity a ON s.activity_id = a.id
    WHERE s.id = #{id}
</select>
```

## 📊 关联映射深度分析

### 关联映射的优势

**问题场景**：需要获取每个报名的用户和活动信息

**❌ 传统JDBC方式**：
```java
// 需要多次数据库查询
Signup signup = dao.getSignup(1);           // SQL查询1
User user = dao.getUser(signup.getUserId()); // SQL查询2
Activity activity = dao.getActivity(signup.getActivityId()); // SQL查询3
// 手动组装数据，容易出错
```

**✅ MyBatis关联映射**：
```java
// 只需一次数据库查询
Signup signup = mapper.getSignupById(1);    // SQL查询1 (包含JOIN 2个表)
// 自动组装数据
String userName = signup.getUser().getUsername();      // 直接获取
String activityTitle = signup.getActivity().getTitle(); // 直接获取
```

### 性能对比

| 方式 | 数据库查询数 | 代码复杂度 | 代码量 |
|-----|-----------|---------|-------|
| 传统JDBC | 3次 | 高 | 多 |
| MyBatis关联映射 | 1次 | 低 | 少 |

## 🔍 关键技术总结

### MyBatis 关联映射的三种处理方式

在本项目中实现的关联映射类型：

#### 1. 一对一关联（Association）

```xml
<!-- in SignupMapper.xml -->
<association property="user" ofType="User" select="..."/>
<!-- or -->
<association property="activity" ofType="Activity">
    <result column="a_id" property="id"/>
    <!-- ... -->
</association>
```

#### 2. 关键SQL - 使用LEFT JOIN进行多表查询

```sql
SELECT
    s.id s_id, s.user_id s_user_id, ...,
    u.id u_id, u.username u_username, ...,
    a.id a_id, a.title a_title, ...
FROM signup s
LEFT JOIN `user` u ON s.user_id = u.id
LEFT JOIN activity a ON s.activity_id = a.id
WHERE s.id = 1
```

#### 3. 结果集映射 - 通过列别名区分不同表的数据

